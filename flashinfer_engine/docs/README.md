# Documentation Guide

This folder has two kinds of documents:

- active docs: the files that define the current project direction
- archived docs: older notes kept only for reference

## Active Docs

- `PROJECT_ROADMAP.md`: the scoped, resume-oriented roadmap
- `EXECUTION_PLAN.md`: the step-by-step plan we will follow
- `ARCHITECTURE.md`: current code structure and data flow
- `PERFORMANCE.md`: current benchmark snapshot
- `ENVIRONMENT.md`: environment setup notes

## Archived Docs

Historical notes live under `archive/`.

- `archive/INTERVIEW_SUMMARY.md`
- `archive/OPTIMIZATION_PLAN.md`
- `archive/PROFILE_REPORT.md`

These files are not the source of truth for the current plan.
They may contain stale references to older roadmap files or older doc paths.

## Source Of Truth

If documents disagree:

- `EXECUTION_PLAN.md` decides what we do next
- `PROJECT_ROADMAP.md` defines the intended scope
